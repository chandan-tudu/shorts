Python Common Built-in Exceptions

In Python, exceptions are raised when an error or unexpected situation occurs during the execution of a program. Each type of exception is represented by a specific class, and these classes are organized in a hierarchy. Here are some common built-in exceptions in Python:

  1. SyntaxError:

    • Raised when there is a syntax error in the code.
    print("Hello, World"  # Missing closing parenthesis

  2. IndentationError:

    • Raised when there is an indentation error in the code.
    def example():
print("Indented incorrectly")

  3. NameError:

    • Raised when a local or global name is not found.
    print(undefined_variable)

  4. TypeError:

    • Raised when an operation or function is applied to an object of an inappropriate type.
    result = "10" + 5

  5. ZeroDivisionError:

    • Raised when division or modulo by zero occurs.
    result = 10 / 0

  6. ValueError:

    • Raised when a built-in operation or function receives an argument of the correct type but an inappropriate value.
    number = int("abc")

  7. IndexError:

    • Raised when a sequence subscript is out of range.
    my_list = [1, 2, 3]
element = my_list[5]

  8. FileNotFoundError:

    • Raised when an attempt is made to open a file or directory that cannot be found.
    with open("nonexistent_file.txt", "r") as file:
   content = file.read()

  9. KeyError:

    • Raised when a dictionary key is not found.
    my_dict = {"name": "John"}
value = my_dict["age"]

  10. AttributeError:

    • Raised when an attribute reference or assignment fails.
    value = "Hello, World"
length = value.length  # AttributeError: 'str' object has no attribute 'length'

These are just a few examples of built-in exceptions in Python. Understanding the types of exceptions and their meanings helps in debugging and handling errors more effectively in your Python programs. You can catch and handle exceptions using try, except, else, and finally blocks.
