Python @property Decorator With Example

In Python, the @property decorator is used to create read-only properties for class instances. It allows you to define a method that can be accessed like an attribute, without the need for parentheses when calling it.

The @property decorator is often used to encapsulate the access and modification of private instance variables in a class.

Here's a simple example to illustrate the use of the @property decorator:

class Circle:
    def __init__(self, radius):
        self._radius = radius

    @property
    def radius(self):
        return self._radius

    @property
    def diameter(self):
        return 2 * self._radius

    @property
    def area(self):
        return 3.14 * self._radius ** 2

# Example usage
circle = Circle(5)

print(circle.radius)    # Accessing property like an attribute
print(circle.diameter)  # Accessing another property
print(circle.area)      # Accessing another property

In this example:

  • The Circle class has a private instance variable _radius, which is conventionally considered private by using a single leading underscore.

  • The @property decorator is applied to three methods: radius, diameter, and area. These methods act as read-only properties and can be accessed like attributes.

  • When accessing circle.radius, circle.diameter, or circle.area, the corresponding method is called, and the value is computed and returned.

  • Since these properties are read-only, attempting to set them will raise an AttributeError.

# Attempting to set a read-only property will raise an AttributeError
circle.radius = 7  # Raises AttributeError

The @property decorator can also be combined with the @property_name.setter decorator to create read-write properties with associated setter methods. This allows for controlled access and modification of the underlying private attributes.

class Square:
    def __init__(self, side_length):
        self._side_length = side_length

    @property
    def side_length(self):
        return self._side_length

    @side_length.setter
    def side_length(self, value):
        if value > 0:
            self._side_length = value
        else:
            raise ValueError("Side length must be positive")

    @property
    def area(self):
        return self._side_length ** 2

# Example usage
square = Square(4)

print(square.side_length)  # Accessing property like an attribute
print(square.area)         # Accessing another property

square.side_length = 5     # Modifying the property using the setter
print(square.side_length)  # Output: 5
print(square.area)         # Output: 25

In this example, the Square class has a read-write property side_length with an associated setter method. The setter method checks that the assigned value is positive before allowing the modification.

The @property decorator is a powerful tool for creating properties that provide controlled access to underlying attributes, making the code more readable and maintainable.
