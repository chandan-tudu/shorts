Python Encapsulation (Private And Protected)

Encapsulation is one of the fundamental principles of object-oriented programming (OOP). It refers to the bundling of data (attributes) and the methods (functions) that operate on the data within a single unit, known as a class. The purpose of encapsulation is to restrict access to some of an object's components and to prevent the accidental modification of data.

In Python, encapsulation is achieved through the use of private and protected access modifiers and property decorators. Here's an overview:

Private and Protected Attributes:

  1. Private Attributes:

    • Private attributes are indicated by using a double underscore (__) before the attribute name. These attributes are not directly accessible from outside the class.
    class Car:
   def __init__(self, make, model):
       self.__make = make   # Private attribute
       self.__model = model

   def display_info(self):
       print(f"{self.__make} {self.__model}")

my_car = Car("Toyota", "Camry")
my_car.display_info()
# Accessing private attributes directly will result in an AttributeError
# print(my_car.__make)  # This line will raise an error

  2. Protected Attributes:

    • Protected attributes are indicated by using a single underscore (_) before the attribute name. While they can be accessed from outside the class, it is considered a convention to treat them as non-public.
    class Person:
   def __init__(self, name, _age):
       self.name = name
       self._age = _age   # Protected attribute

person = Person("Alice", 25)
print(person.name)     # Accessible
print(person._age)      # Accessible, but consider it non-public

Property Decorators:

In Python, you can use property decorators to create getter and setter methods for attributes, providing controlled access to them.

class Circle:
    def __init__(self, radius):
        self.__radius = radius   # Private attribute

    @property
    def radius(self):
        return self.__radius

    @radius.setter
    def radius(self, value):
        if value > 0:
            self.__radius = value
        else:
            print("Radius must be greater than 0.")

# Example usage
circle = Circle(5)
print(circle.radius)  # Accessing the attribute using the getter method
circle.radius = 8     # Setting the attribute using the setter method
print(circle.radius)

In this example, the radius property has a getter (@property) and a setter (@radius.setter). This allows controlled access to the private attribute __radius.

Encapsulation helps in preventing accidental modifications to an object's internal state and allows the class to change its internal implementation without affecting the code that uses the class. It promotes information hiding and improves the maintainability of code.
