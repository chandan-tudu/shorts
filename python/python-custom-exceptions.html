Python Custom Exceptions

In Python, you can create custom exceptions by defining a new class that inherits from the built-in Exception class or one of its subclasses. This allows you to raise and catch exceptions that are specific to your application. Here's an example of how to create and use custom exceptions:

class CustomError(Exception):
    """Custom exception class."""
    def __init__(self, message="A custom error occurred."):
        self.message = message
        super().__init__(self.message)

# Example of raising a custom exception
def example_function(value):
    if value < 0:
        raise CustomError("Negative values are not allowed.")

try:
    user_input = int(input("Enter a positive number: "))
    example_function(user_input)
except CustomError as ce:
    print(f"Custom Error: {ce}")
except ValueError as ve:
    print(f"Value Error: {ve}")
else:
    print("No exceptions were raised.")

In this example:

  1. CustomError is a custom exception class that inherits from the built-in Exception class. It has an optional constructor that allows you to provide a custom error message.

  2. The example_function function raises a CustomError if the provided value is negative.

  3. In the try block, the user is prompted to enter a positive number. If the entered value is negative, a CustomError is raised. If the entered value is not a valid integer (resulting in a ValueError), that exception is caught as well.

  4. The except blocks handle the custom exception (CustomError) and the built-in exception (ValueError). If neither exception occurs, the else block is executed.

Custom exceptions are useful when you want to raise errors that are specific to your application's logic or when you want to create a hierarchy of exceptions to handle different error scenarios more precisely.
