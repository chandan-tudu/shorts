Python Variable Scope

In Python, variable scope refers to the region of the code where a variable can be accessed or modified. Understanding variable scope is important for writing clear and error-free code. In Python, there are two main types of variable scope: global scope and local scope.

  1. Global Scope:

    • Variables defined outside of any function or block have a global scope. They can be accessed from any part of the code, including within functions.
    global_variable = 10

def print_global():
   print(global_variable)

print_global()  # Output: 10

    In this example, global_variable has global scope and can be accessed both inside and outside the print_global function.

  2. Local Scope:

    • Variables defined inside a function have a local scope. They are only accessible within that function.
    def print_local():
   local_variable = 5
   print(local_variable)

print_local()  # Output: 5

# Attempting to access local_variable outside the function would result in an error

    In this example, local_variable is defined within the print_local function and can only be accessed within that function.

    def modify_global():
   global global_variable
   global_variable = 20

modify_global()
print(global_variable)  # Output: 20

    Note: If you want to modify a global variable within a function, you can use the global keyword to indicate that you are working with the global variable.

  3. Enclosing Scope (Nonlocal):

    • In nested functions, a variable defined in the outer (enclosing) function can be accessed in the inner function if the nonlocal keyword is used.
    def outer_function():
   outer_variable = 15

   def inner_function():
       nonlocal outer_variable
       print(outer_variable)

   inner_function()

outer_function()  # Output: 15

    In this example, outer_variable is defined in the outer function and accessed within the inner function using the nonlocal keyword.

Understanding variable scope is essential for avoiding naming conflicts and ensuring that variables are used in the appropriate context. It helps in writing modular and maintainable code.
