Python Exceptions

In Python, exceptions are events that occur during the execution of a program that disrupt the normal flow of the program's instructions.

When an exceptional situation arises, an exception object is raised, and the normal flow of the program is interrupted. To handle exceptions, you use a mechanism called "exception handling."

Here's a basic overview of how exception handling works in Python:

The try, except Block:

You use the try and except blocks to handle exceptions. Code that might raise an exception is placed inside the try block, and the handling of the exception is specified in the except block.

try:
    # Code that might raise an exception
    result = 10 / 0  # This will raise a ZeroDivisionError
except ZeroDivisionError:
    # Code to handle the exception
    print("Error: Division by zero")

Handling Multiple Exceptions:

You can handle multiple exceptions by providing multiple except blocks.

try:
    value = int(input("Enter a number: "))
    result = 10 / value
except ZeroDivisionError:
    print("Error: Division by zero")
except ValueError:
    print("Error: Invalid input. Please enter a number.")

The else and finally Blocks:

  • The else block is executed if no exceptions are raised in the try block.
try:
    result = 10 / 2
except ZeroDivisionError:
    print("Error: Division by zero")
else:
    print("Result:", result)
  • The finally block is executed no matter what, whether an exception occurred or not. It is often used for cleanup operations.
try:
    value = int(input("Enter a number: "))
    result = 10 / value
except ZeroDivisionError:
    print("Error: Division by zero")
finally:
    print("This will be executed no matter what.")

Raising Exceptions:

You can also explicitly raise exceptions using the raise statement.

def example_function(value):
    if value < 0:
        raise ValueError("Value must be non-negative")
    return value * 2

try:
    result = example_function(-5)
except ValueError as e:
    print(f"Caught an exception: {e}")

Understanding and handling exceptions is crucial for writing robust and reliable Python programs. It allows you to gracefully handle errors and prevent unexpected crashes.
