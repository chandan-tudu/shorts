Python "global" Keyword

In Python, the global keyword is used inside a function to indicate that a variable defined within that function is a global variable, rather than a local variable.

This means that the variable is not confined to the function's scope but is instead accessible and modifiable globally within the entire program.

Here's a simple example to illustrate the use of the global keyword:

global_variable = 10  # This is a global variable

def modify_global_variable():
    global global_variable  # Declare that we want to use the global variable within the function
    global_variable += 5    # Modify the global variable inside the function

modify_global_variable()

print("Global variable after modification:", global_variable)

In this example, the modify_global_variable function uses the global keyword to indicate that it wants to modify the global_variable defined outside the function.

Without the global keyword, Python would create a new local variable with the same name inside the function, and any changes would only affect that local variable, not the global one.

It's worth noting that the use of global variables is generally discouraged because it can make the code harder to understand and maintain.

Instead, it's often better to use function parameters and return values to pass data between functions. Global variables should be used sparingly and only when necessary.
